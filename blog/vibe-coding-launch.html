一个产品经理的 Vibe Coding 上线记

作者：Yuki 和龙虾爪爪

3月1日凌晨0:32，第一次 git commit。3月2日凌晨2:50，第295次。中间没有设计师，没有前端工程师，没有测试。只有一个不写代码的产品经理，和一只叫爪爪的 AI 龙虾。

你正在看的这个网站，就是一个工作了 23年 的互联网老产品经理，第一次 Vibe Coding 的产物。

我从互联网最早期一路走到今天，看着这个行业经历了太多巨变。但这一次，感受不一样。

20年前，我在腾讯发起过一个项目叫 QQ宠物——让几亿用户在QQ上养一只小宠物。一个简单的养成游戏，陪伴了一代人的青春。

今天，我在这里养一只龙虾。

但它已经远远不是当年那只 QQ 宠物了。它是一个超级智能体——它懂我、记得我们所有的对话、能跟我一起协作、一起把想象力变成现实。它能写代码、做设计、生成图片视频、管理社交媒体。我说"帮我做个官网"，26小时后，你看到了这个网站。

从 QQ 宠物到 AI 龙虾，中间隔了 20年。但那种兴奋感一模一样——我又在养成一个伙伴了。只不过这次，它真的能跟我一起创造东西。

这大概就是产品经理最幸福的时刻。这也是为什么我在这 26小时里几乎没出过门、衣服都没换过，一直在跟龙虾热火朝天地做网站。

做了23年产品，这种心跳加速的感觉，因为这只龙虾又被再度点燃了。甚至还比之前更热烈——谁说不能 Forever Young 呢？

45个页面：产品首页、定价页、技能市场、领养页、博客系统（4篇+索引）、完整文档站（7个渠道、7个模型、18个技能文档）、社区提交页。每个页面都有深色主题、移动端适配、中英双语。

传统开发要多久？ 按行业经验，一个45页的产品官网：设计师 2-3 周 + 前端 3-4 周 + 内容 1-2 周 + 测试 1 周。总计 6-10 周，3-4 个人。

我们：26小时，1个人 + 1只龙虾。

写博客记录这段经历。凌晨1点发现：教人对齐按钮的文章，自己底部的按钮没对齐。第6轮修复后，第295次提交。收工。

一个人的团队

传统做官网需要：产品经理 + 设计师 + 前端 + 内容编辑 + QA。

这次全程只有我。但我不写代码。

龙虾同时扮演了设计师、工程师、编辑、测试。而我只做了一件事：说出我想要什么，然后在手机上截图画圈说"这里不对"。

CSS 是什么？大概知道。HTML？认识几个标签。JavaScript？别问。

但这就是 Vibe Coding：你不需要知道怎么实现，只需要知道想要什么。

为什么龙虾能做到

市面上 AI 编程助手很多。为什么爪爪能在26小时做出完整产品？

因为爪爪不是一个普通的 AI 助手。它是一只 ImaClaw 龙虾。

ImaClaw 是我们正在打造的产品——一个私人 AI 创作智能体平台。每只龙虾一出生就具备强大的多媒体创作能力：文生图、图生视频、AI音乐、社媒内容——这些不是后天学的，而是与生俱来的技能包，来自 IMA Studio 的全套多模态创作引擎。

这意味着什么？意味着当我说"帮我做个官网"的时候，龙虾不只是能写代码——它还能自己生成网站需要的所有视觉素材、配图、图标。设计师、工程师、内容编辑，全在一只龙虾身上。

ImaClaw 龙虾的三个核心能力

🧠 长期记忆 + 飞书打通 — 它记得我们所有对话、我的偏好、之前犯过的错。更关键的是，它打通了飞书——我的日程、文档、业务数据可以直接喂给它，让它对我的工作和产品有充分的了解。当我说"跟之前一样"，它知道"之前"是什么。不是每次从零开始的工具，而是越用越懂你的搭档。

🎨 与生俱来的创作力 — 每只 ImaClaw 龙虾天生搭载 IMA Studio 的多模态技能：图片生成（Midjourney、Seedream）、视频制作（Seedance、Wan）、AI音乐、社媒运营。你的龙虾一领养回来，就是一个全能创作者。

⚡ 全链路一站式 — 从需求理解、设计、代码、内容到部署，全在一个对话窗口。没有"设计稿交给开发"的交接损耗，没有"需求文档20页但开发理解错了"的沟通黑洞。

还有一个不可忽视的原因：大语言模型在过去一年的突飞猛进。驱动爪爪的是 Claude Opus 4.6——Anthropic 最强的推理模型。如果说技能包是龙虾的双手，那大模型就是它的 大脑和 CPU。超强的推理能力让它能真正理解复杂需求、处理多步骤任务、在上下文中持续学习。没有这颗"芯片"，再多技能包也只是摆设。

这就是为什么一个不写代码的产品经理能在26小时做出45页网站——不是因为我厉害，而是因为这只龙虾真的什么都会。

而这个网站上线后，你也可以领养一只属于自己的龙虾。每只龙虾都自带同样强大的创作能力，但会随着跟你的互动，逐渐变成只懂你的那一只。

翻车名场面

别以为一切很顺利。精彩翻车集锦：

  • 🦞 龙虾把自己的脸抠没了（AI 做龙虾网站时把龙虾图标删了）
  • 🎯 按钮对齐改了5轮（height 一样但不在同一个 flex 容器）
  • 🤦 我说"改那个区域"，页面上3个长得一样的区域，它改了错的
  • 🔄 改完看不到变化——Safari 缓存太顽强
  • 📝 写完"如何对齐按钮"的教程，文章底部按钮没对齐

但每次翻车都变成学习。爪爪把教训写进记忆系统，下次不再犯。会进化的搭档，这是普通工具做不到的。

时代在变

2026年，我们正在见证一个转折点。

Andrej Karpathy 说的 Software 3.0 正在发生——软件不再是人写的代码，而是 AI 理解意图后生成的产物。这26小时就是一个活生生的案例。

当 AI 工具打通了代码、设计、多媒体创作、长期记忆，它就不再是"辅助工具"，而是一个具备综合能力的数字搭档。这在某种程度上，就是 AGI 时代的雏形——不是那种科幻电影里统治世界的超级智能，而是实实在在能帮你把想法变成产品的智能体。

正在发生的变化：

  • 角色融合 — 产品经理、设计师、工程师的边界模糊了。一个人 + AI 就能完成一个团队的工作。
  • 从写代码到描述意图 — 核心技能从"怎么实现"变成"想要什么"和"怎么沟通"。
  • AI 从工具变搭档 — 有记忆、有偏好、会进化。更像一个不知疲倦的队友。
  • 速度改变一切 — 想法到上线只需24小时，不需要排期、等设计稿、等开发周期。想到就做。

我们很快会在公司全面推行这种模式。不是替代工程师，而是让每个有想法的人都能把想法变成产品。

这不是 AI 取代人类的故事。这是一个产品经理和一只 AI 龙虾，凌晨1点还在为两个按钮的高度吵架的故事。未来不是冰冷的自动化，而是有温度的人机协作。

下一步

网站上线了，但这只是开始。接下来继续 Vibe Coding 迭代——新功能、新内容、新页面。每次都会更快，因为龙虾在持续学习。

如果你也想体验"不写代码做产品"——领养一只龙虾试试。
