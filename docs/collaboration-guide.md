## 1️⃣ 截图标注法（最有效！）

你现在已经用得很好了——**截图 + 画圈**。可以更进一步：

### 标注颜色约定
- 🟡 **黄色圈** = "改这里"（你现在用的）
- 🔴 **红色叉** = "删掉"
- 🟢 **绿色箭头** = "移到这里"

### 描述模板
改之前说三件事就够：

> **哪里** + **怎么改** + **改成什么**

✅ 好的例子：
- "导航栏的汉堡菜单 → 去掉边框"
- "这个按钮（截图）→ 改成蓝色"
- "这两个按钮 → 高度要一样"

❌ 容易误解的：
- "这个区域改一下"（哪个区域？改什么？）
- "不太好"（哪里不好？间距？颜色？字体？）

## 2️⃣ 10个必知的网页术语

这些词我们天天用，学会后沟通效率翻倍：

### 布局类

| 术语 | 意思 | 你可以说 |
|------|------|---------|
| **导航栏 / Nav** | 顶部那一条（logo + 菜单） | "导航栏的间距太大" |
| **Hero** | 页面最顶部的大标题区域 | "Hero 标题换一下" |
| **卡片 / Card** | 一个有边框的内容块 | "这个卡片删掉" |
| **页脚 / Footer** | 页面最底部的区域 | "页脚加个链接" |
| **汉堡菜单** | 手机上的 ☰ 三条杠按钮 | "汉堡菜单样式不对" |

### 样式类

| 术语 | 意思 | 你可以说 |
|------|------|---------|
| **间距 / 边距** | 元素之间的空白距离 | "两个按钮间距太大" |
| **内边距 (Padding)** | 内容到边框的距离 | "卡片内边距太小，文字贴边了" |
| **外边距 (Margin)** | 元素和外面其他东西的距离 | "这个区块离上面太近" |
| **圆角** | 方框的角是不是圆的 | "按钮圆角大一点" |
| **透明度** | 颜色的透不透 | "背景透明度低一点（更实）" |

### 快速定位词

| 你说 | 我理解 |
|------|--------|
| "往上/下/左/右挪" | 调 margin 或 position |
| "放大/缩小" | 调 font-size 或 width/height |
| "颜色浅一点/深一点" | 调 color/opacity |
| "松一点/紧一点" | 调 padding 或 gap |
| "对齐" | 调 text-align 或 flex 布局 |
| "换行不好看" | 调 line-break 或缩短文字 |

## 3️⃣ 缓存问题速解

今天最大的时间浪费就是缓存！你看到的经常是旧版本。

### 解决方法
每次我改完会给你一个带 `?v=数字` 的链接，比如：
```
https://imaclaw.github.io/ima-claw/pricing/?v=207
```

**规则：永远用我给的最新链接打开**，不要直接刷新页面。

### 如果还是旧的
iPhone Safari 强制刷新：
1. 长按地址栏的刷新按钮 → "请求桌面网站"再切回来
2. 或者：设置 → Safari → 清除历史记录和网站数据（核弹级）

## 4️⃣ 我承诺的改进

### 改之前先确认
以后遇到模糊指令，我会先确认再动手：
> "你说的是 [具体位置] 的 [具体元素] 吗？要改成 [具体效果]？"

### 一次改对
不再一个属性一个属性地试，我会：
1. 先完整分析问题（对比参考页面）
2. 一次性改全（CSS + HTML + 移动端）
3. 自测通过再推送

### 每次推送附带改动说明
> "✅ 改了3处：① 按钮高度统一44px ② 加了--blue变量 ③ 汉堡菜单去边框"

这样你一眼知道改了什么，不用猜。

## 5️⃣ 高效反馈模板

如果你想快速给反馈，用这个模板：

### 简单改动
> 📍 [页面] [位置]  
> 🔧 [改成什么]

例如：
> 📍 pricing 页 Hero 标题  
> 🔧 "6个" 改成 "多个"

### 复杂改动
> 📍 [页面] [截图+画圈]  
> 🔧 [要达到的效果]  
> 📎 [参考：某个页面/某个网站]

例如：
> 📍 skills 页底部（截图）  
> 🔧 两个按钮一样大，跟首页的按钮风格一致  
> 📎 参考首页的"立即预约"按钮
